1. Introducción a PHP
1.1 ¿Qué es PHP?

PHP (acrónimo recursivo para PHP: Hypertext Preprocessor) es un lenguaje de programación de código abierto, interpretado y de uso general, diseñado especialmente para el desarrollo web del lado del servidor. Fue creado originalmente por Rasmus Lerdorf en 1994 y hoy en día es el motor de más del 75% de los sitios web del mundo, incluyendo plataformas masivas como Wikipedia, WordPress y los inicios de YouTube.

1.2 En el mundo Web: PHP embebido en HTML

A diferencia de lenguajes como C, donde necesitas usar una gran cantidad de comandos como printf() para estructurar una salida de texto, PHP nos permite incrustar o embeber código directamente dentro de un archivo HTML utilizando las etiquetas especiales de procesamiento: <?php y ?>.

Veamos un ejemplo:

        
        
<!DOCTYPE html>
<html>
   <head>
       <title>Ejemplo Introductorio</title>
   </head>

   <body>
    
       <!-- Esto es HTML puro -->
       <h1>Página de Bienvenida</h1>
    
       <?php
            // Esto es código PHP ejecutándose dentro del HTML
            echo "¡Hola, soy un script de PHP!";
       ?>
    
   </body>
</html>
1.3 Ciclo de Vida de una Petición (El Lado del Servidor)

A diferencia de C, donde el binario compilado se ejecuta localmente en la máquina del usuario, PHP opera bajo el modelo de arquitectura Cliente-Servidor. En este modelo el flujo de ejecución sigue el orden: 

        
        
[   Cliente: Navegador   ] ------- Petición HTTP (URL) ---------> [ Servidor Web ]
                                                                          |
                                                                          v
[ Cliente: Renderiza HTML ] <-- Respuesta HTTP (HTML Puro) <-- [ Intérprete PHP (Zend) ]

El proceso se divide en tres etapas de bajo nivel:

  1. Entrada (Request): El cliente realiza una petición HTTP solicitando un archivo .php.
  2. Procesamiento (Parsing): El servidor web transfiere el archivo al intérprete de PHP. Este ejecuta la lógica, evalúa las estructuras de control y procesa los datos necesarios.
  3. Salida (Response): El intérprete descarta todo el código fuente de PHP y entrega exclusivamente el flujo resultante (generalmente strings de HTML, JSON o binarios de imágenes) al servidor, el cual lo regresa al cliente como una respuesta HTTP estándar.

Nota: El cliente jamás tiene acceso al código fuente de PHP (las credenciales de bases de datos, algoritmos o consultas SQL se quedan protegidas en el servidor), manteniendo la misma seguridad que un ejecutable en C.

1.4 Capacidades y aplicaciones

Aunque su aplicación principal es en el desarrollo web (gestión de formularios, generación de contenido dinámico, manejo de cookies, etc...), PHP es sumamente versátil:

  • Multiplataforma y Servidores: Funciona de forma gratuita en casi todos los sistemas operativos (Linux, Windows, macOS) y servidores web (Apache, Nginx, IIS).
  • Soporte Masivo de Bases de Datos: Permite conexiones nativas y sencillas a sistemas SQL y NoSQL como MySQL, PostgreSQL, SQLite, MariaDB, MongoDB, entre otros.
  • Más allá de la Web: Cuenta con una interfaz de línea de comandos (PHP-CLI) ideal para scripts automatizados (tareas cron), procesamiento de texto e incluso desarrollo de aplicaciones gráficas autónomas (PHP-GTK) o control de drones.
  • Paradigmas: Es multiparadigma entonces nos permite decidir entre programar de manera estructurada (como en C) o programación orientada a objetos (OOP).
2. Entorno de Desarrollo (Instalación con XAMPP)
2.1 ¿Cómo empiezo a desarrollar con PHP?

Al momento de desarrollar proyectos con lenguaje C, el flujo es directo: el compilador (GCC) genera un archivo binario ejecutable que el Sistema Operativo aloja en la memoria RAM para que el procesador lo ejecute directamente a nivel hardware.

Sin embargo, el flujo cambia por completo con PHP y la arquitectura web. PHP no se ejecuta bajo demanda directa del usuario o del sistema operativo local, sino como respuesta a una petición HTTP remota. Una computadora común no tiene activas las herramientas para gestionar este protocolo de red ni para comunicarse con un navegador web. Para poder programar y realizar pruebas de manera local, es por esto que necesitamos instalar un software que simule en nustra computadora el entorno de un servidor web real.

Según la documentación oficial de PHP, para el desarrollo de aplicaciones web (Server-side scripting) se requiere la interacción mandatoria de tres componentes independientes:

  • El Intérprete (PHP mismo): El motor que compila, procesa y ejecuta la lógica del código backend.
  • Un Servidor Web: El software encargado de escuchar la red, recibir las peticiones HTTP y regresar las respuestas.
  • Un Navegador Web: El cliente encargado de enviar las peticiones y renderizar el resultado final (como Chrome, Firefox o Edge).

2.2 ¿Cómo interactuan estos componentes entre sí?
  1. Petición de Entrada (HTTP Request): El Cliente (Navegador Web) solicita un recurso a través de una URL. Esta petición viaja por la red buscando un puerto abierto en el servidor.
  2. Intercepción del Servidor: El servidor web escucha la petición en su puerto. Como el sistema operativo no entiende el protocolo HTTP de forma nativa, este servidor web actúa como el receptor y administrador de la red.
  3. Enrutamiento por Interfaz: Al detectar que el archivo solicitado tiene extensión .php, el servidor web utiliza la interfaz de módulo SAPI (Server Application Programming Interface) o un procesador CGI para desviar el archivo hacia el intérprete, en lugar de enviarlo directamente al cliente.
  4. Procesamiento y Compilación (Zend Engine): El intérprete de PHP procesa el script, ejecuta la lógica (ciclos, condicionales, peticiones a la base de datos) y compila el resultado final, convirtiendo todo el bloque dinámico en un flujo de datos plano (HTML, JSON o binarios).
  5. Retorno de Salida (HTTP Response): El intérprete le regresa el flujo de datos limpio a servidor web, que empaqueta ese resultado dentro de una respuesta HTTP y se la envía de vuelta al navegador web del cliente para que lo renderice.
2.3 Solución en un solo paso: XAMPP

Configurar el Servidor Web, el Intérprete (PHP) y el Sistema de Base de Datos de forma nativa e independiente requeriría modificar manualmente archivos de configuración del sistema operativo y gestionar permisos de red complejos.

Para resolver esto de forma eficiente en el entorno de desarrollo local, utilizaremos XAMPP. Este software empaqueta las tres piezas obligatorias en una arquitectura pre-configurada que se ejecuta con un solo clic.

XAMPP es un paquete de software libre y gratuito que incluye Apache, MariaDB, PHP y Perl, lo que le permite ofrecer todos los componentes necesarios para desarrollar aplicaciones web con total garantía en un entorno local. De hecho, su nombre es en realidad un acrónimo que refleja su naturaleza multiplataforma (X), y los componentes clave que integra: Apache (A), MariaDB (M), PHP (P) y Perl (P).

3. Tu primer programa en PHP
3.1 ¿Dónde escribir el código?

A diferencia de C, donde guardas tus archivos .c en cualquier carpeta de tus documentos y los compilas ahí mismo, en PHP el servidor Apache solo puede ejecutar archivos que estén dentro de su directorio raíz público.

Crea una carpeta llamada PHP/Ejemplos dentro de la ruta que te corresponde según tu sistema operativo:

  • Windows: C:\xampp\htdocs\CW-2026\PHP\Ejemplos/
  • Linux: /opt/lampp/htdocs/CW-2026/PHP/Ejemplos/
  • macOS: /Applications/XAMPP/htdocs/CW-2026/PHP/Ejemplos/
3.2 Creando el archivo

Ya sea usando comandos en terminal o tu editor de código preferido, crea un archivo nuevo en la ruta del punto anterior, y nombralo como holaMundo.php.

3.3 El código

En PHP no necesitamos importar bibliotecas de entrada/salida ni declarar un método main(). El intérprete ejecuta el archivo de forma secuencial desde que encuentra la etiqueta de apertura, veamos una comparativa:

Estructura en C (holaMundo.c) Estructura en PHP (holaMundo.php) 
           
             
#include <stdio.h>
         
int main() {
    printf("¡Hola Mundo desde C!\n");
    return 0;
}
                        
           
           
             
<?php         

    echo "¡Hola Mundo desde PHP!";

?>
3.4 Despligue en el navegador

Para ver el resultado de tu programa, asegúrate de que el módulo Apache esté encendido en tu panel de XAMPP. Abre tu navegador web y accede a la siguiente dirección URL:

http://localhost/CW-2026/PHP/Ejemplos/holaMundo.php
3.5 ¿Qué pasa cuando usamos localhost?

Una de las mayores diferencias al programar en PHP respecto a C es que el código no se ejecuta directamente en tu pantalla. PHP vive y muere en un Servidor Web. Cuando usas XAMPP en tu computadora, tu computadora juega un "doble rol": es el cliente que pide la página y el servidor que ejecuta lo necesario.

Para entender cómo se conecta PHP con los conceptos de redes que ya conoces, veamos el viaje que hace tu computadora cada vez que escribes http://localhost/CW-2026/PHP/Ejemplos/holaMundo.php en el navegador:

  1. DNS (El Atajo Local): En el mapa general, el DNS busca la IP del servidor en el mundo. Pero aquí, al escribir localhost, tu sistema operativo no sale a internet. Tiene un archivo interno que le dice de inmediato: "localhost equivale a la IP 127.0.0.1". Esta IP especial significa "yo mismo". Es decir el viaje es hacia nosotros mismos.

  2. HTTP (La Orden del Cliente): Tu navegador (en su rol de Cliente) prepara el mensaje y le pone las etiquetas de petición. Dice: GET /CW-2026/PHP/Ejemplos/holaMundo.php HTTP/1.1. Básicamente, es como la comanda con el platillo que el cliente quiere ordenar.

  3. TCP (El Canal Seguro): Para que el mensaje no se pierda ni se rompa, TCP lo empaca de forma segura. En tu computadora, TCP abre un "tubo virtual" interno directo hacia el Puerto 80, que es la puerta donde el servidor Apache de XAMPP está sentado esperando peticiones.

  4. IP (La Guía de Ruta): Pone las direcciones de origen y destino. En este caso, el origen es tu máquina (127.0.0.1) y el destino también es tu máquina (127.0.0.1). Al ver esto, la arquitectura de red sabe que no debe mandar el paquete a los routers de tu proveedor de internet, sino desviarlo internamente hacia la XAMPP.

    Aquí es donde Apache recibe el paquete, ve que es un archivo .php, el motor de PHP ejecuta y genera el HTML final con los echo y printf.

  5. Flujo Físico (El Viaje Interno): En internet, los datos viajarían como luz por fibra óptica o electricidad por cables submarinos. En localhost, el flujo físico ocurre a través de los circuitos de tu memoria RAM y tu tarjeta de red virtual. La información se mueve a la velocidad de tu procesador para entregarle la respuesta de vuelta al navegador.
4. Sintaxis Esencial y Reglas del Lenguaje
4.1 Etiquetas de Procesamiento

El intérprete de PHP solo procesa el código que se encuentra entre las etiquetas <?php y ?>. Todo lo que esté fuera de ellas se envía al navegador como HTML plano.

4.2 Escapar de HTML

Puedes abrir y cerrar bloques PHP cuantas veces quieras en un mismo archivo para intercalar lógica y diseño: 

<h1>Título en HTML</h1>
    
<?php
    echo "<p>Este párrafo se generó en el servidor.</p>";
?>
    
<p>Este es otro texto fijo en HTML.</p>

<?php
    echo "<p>Este otro se generó en el servidor.</p>";
?>
4.3 ¿Qué onda con el main()?

En C, tu código está obligado a vivir dentro de una función estructurada llamada int main(). Si no existe, el programa simplemente no compila.

En PHP no existe el main(). PHP es un lenguaje de scripting, lo que significa que el servidor lee el archivo de arriba hacia abajo y va ejecutando cada línea de instrucción de forma secuencial en cuanto encuentra la etiqueta <?php. Tu archivo completo es el flujo principal.

4.4 Declaración, Asignación y Tipado Dinámico

En C tenías que avisarle al compilador qué caja de memoria querías reservar, es decir, declaración (int x;) y luego, si querías, le dabas un valor, es decir asignación (x = 5;). Además, esa variable se mantenía del mismo tipo durante todo el programa.

En PHP creas y asignas en una sola línea. No existen palabras clave para tipos de datos primitivos; el tipo se define automáticamente dependiendo de lo que le metas dentro a la variable, y muta si le cambias el valor.

¿Cómo era en C? (Tipado Estático) ¿Cómo es en PHP? (Tipado Dinámico) 

// 1. Declarar tipo y espacio
int edad;

// 2. Asignar
edad = 21; 

// Esto es un error
// edad = "Veintiuno"; 
<?php
	      
// Se declara y asigna
// al mismo tiempo con $
// Detecta automáticamente que es un entero
$edad = 21; 

// Es perfectamente válido cambiar
// el tipo de dato después
// Detecta que ahora es una cadena
$edad = "Veintiuno";
?>
4.5 Separación de Instrucciones y Comentarios

Al igual que en C, cada instrucción debe finalizar obligatoriamente con un punto y coma (;). Los comentarios usan la misma sintaxis que ya conoces: 

<?php
   // Comentario de una sola línea (Igual que en C)

   /* Comentario de bloque o 
      múltiples líneas (Igual que en C)
   */
?>
4.6 Salida en pantalla

Así como en C usábamos printf() para imprimir en pantalla, en PHP se usa la instrucción echo.

La súper ventaja de PHP: En C necesitabas usar máscaras de formato como %d o %s para concatenar variables. En PHP, si usas comillas dobles (" "), puedes meter tus variables directamente adentro del texto y el intérprete cambiará el nombre de la variable por su valor real (esto se llama interpolación).

Impresión compleja en C Impresión directa en PHP 
int cajas = 5;
char item[] = "manzanas";

// Requiere acordarse de los modificadores
printf("Tengo %d %s.", cajas, item);
 
<?php
$cajas = 5;
$item = "manzanas";

// Las variables se procesan
//directo dentro del string
echo "Tengo $cajas $item."; 
?>

Nota: Si usas comillas simples (' '), PHP no leerá las variables y mandará el texto de forma literal (imprimiría: Tengo $cajas $item).

5. Sistema de Tipos

La documentación oficial dicta que cada expresión en PHP posee un tipo nativo integrado dependiendo de su valor. El sistema de tipos de PHP soporta las siguientes estructuras principales (excluyendo objetos y recursos avanzados):

Tipo ¿Qué representa en PHP? ¿Qué representa en C?
null Es ausencia total e intencionada de valor. Representa una variable sin definir o vacía. NULL es equivalente a un valor entero 0, o a un apuntador que no apunta a ninguna dirección válida.
bool Un valor lógico que solo puede adoptar el estado de true o false. En C tenías que usar enteros (1 o 0) para representarlo o importar la biblioteca stdbool.h.
int Números enteros. Números enteros.
float Números de punto flotante (con decimales). En PHP, un float tiene una precisión equivalente al tipo double de C. Equivale al tipo double de C (precisión doble de 64 bits), ya que el tipo float en C es de precisión simple (32 bits).
string Cadenas de texto. PHP gestiona la memoria automáticamente y permite cualquier tamaño dinámico. No existe el tipo string nativo. Se usa un arreglo de caracteres (char[]) o un apuntador (char*) terminado obligatoriamente en \0.
array Es un mapa (diccionario) ordenado dinámico. Puede almacenar datos de diferentes tipos (heterogéneo) y asociar llaves numéricas o de texto (strings). Es un arreglo estático, de tamaño fijo y de un solo tipo de dato (homogéneo). No permite almacenar tipos mezclados.
6. Variables, Entorno Dinámico y Constantes
6.1 El Identificador Único (El signo $)

En PHP, todas las variables comienzan obligatoriamente con el signo de pesos ($). El nombre de la variable sigue las mismas reglas de C (no puede empezar con números), y sigue la convención de nomenclatura: lowerCamelCase.

Al igual que en C, PHP es case-sensitive, es decir, sensible a mayúsculas y minúsculas. De tal forma que las variables $control y $Control apuntan a espacios de memoria completamente distintos.

6.2 El Concepto de Tipado Dinámico (Runtime)

En C, el tipado es estático: declaramos int x; y esa variable solo guardará enteros para siempre. La documentación oficial de PHP define que es un lenguaje de tipado dinámico, es decir no especificamos el tipo al crear la variable; y el intérprete determina el tipo de dato en tiempo de ejecución (runtime) basándose exclusivamente en el valor que le asignes.

Sabiendo esto, dado el siguiente código, analízalo y determina qué hace:

        
<?php
    $var = 200;    // Asigna un entero (int) a $var
    $Var = "hola"; // Crea una variable distinta (por la V mayúscula) de tipo string
    $var = "hole"; // Reasigna la primera $var cambiándola dinámicamente a string
?>
6.3 Variables Globales

En C, una variable declarada fuera de las funciones es global y cualquier función puede acceder a ella automáticamente. En PHP el "ámbito" (scope) es más estricto: una variable global no es visible dentro de una función a menos que se lo indiques explícitamente usando la palabra clave global.

En C (Acceso directo) En PHP (Requiere declarar 'global') 
int contador = 10;

void funcion() {
    // Se accede directamente
    contador++; 
}
 

<?php
	      
$contador = 10;

function miFuncion() {
    // Debes avisar que usarás la global
    global $contador; 
    $contador++;
}

?>
6.4 Constantes (const)

Las constantes almacenan valores que no pueden cambiar ni borrarse durante la ejecución del programa. A diferencia de las variables, no llevan el signo de pesos ($). Sigue la convención de nomenclatura: SCREAMING_SNAKE_CASE

  • Solo aceptan valores escalares (bool, int, float, string) y arreglos que contengan solo estos tipos.
  • Para verificar si una constante existe, se usa defined().

Ejemplos de uso:


<?php

// 1. Declaración y uso básico

const IMPUESTO = 0.16;
const CONFIG = ["localhost", "root"];
echo IMPUESTO; // Imprime: 0.16 (Sin $)


// 2. Comprobación y lectura dinámica

const MODO = "Oscuro";
$buscar = "MODO";

if (defined($buscar)) 
    echo "Modo definido"; 


?>
7. Operadores
7.1 Precedencia de Operadores

Al igual que en C, la precedencia determina el orden de evaluación de los operadores cuando se encuentran juntos en una expresión (por ejemplo, la multiplicación siempre se ejecuta antes que la suma).

En PHP, al igual que en C, si quieres romper la precedencia natural o asegurarte de que una operation se ejecute primero, debes agrupar la expresión estrictamente entre paréntesis ( ).

7.2 Operadores Aritméticos

Son exactamente iguales a C (excepto la potenciación). Se utilizan para realizar cálculos matemáticos estándar.

Operador Operación Ejemplo
+ Adición 
$res = 5 + 7; // res = 12
- Sustracción 
$res = 7 - 5; // res = 2
* Multiplicación 
$res = 7 * 5; // res = 35
/ División 
$res = 10 / 5; // res = 2
% Módulo (Residuo de la división) 
$res = 10 % 5; // res = 0
** Exponenciación 
$res = 2 ** 3; // res = 2³ = 8
7.3 Incremento y Decremento

Igual que en C, modifican el valor de la variable sumando o restando una unidad. Recuerda la diferencia entre el prefijo y el sufijo.

Notación Operación Efecto
++$var Pre-incremento Incrementa uno a $var y luego regresa $var ya incrementado.
$var++ Post-incremento Regresa $var, luego incrementa uno a $var.
--$var Pre-decremento Decrementa uno a $var y luego regresa $var ya decrementado.
$var-- Post-decremento Regresa $var, luego decrementa uno a $var.

Ejemplos de uso:


// Pre-incremento vs Post-incremento
$var = 5;      
echo ++$var;   // Se incrementa inmediatamente: Imprime 6
echo $var;     // El valor actual es 6

$var = 5;      
echo $var++;   // Regresa el valor original: Imprime 5
echo $var;     // Ahora el valor ya es 6
7.4 Operadores de Bits (Bitwise)

PHP así como C tiene la capacidad de manipular bits individuales dentro de un entero. Se usan los mismos símbolos lógicos a nivel de bit:

Notación Nombre Efecto
$a & $b AND Mantiene los bits prendidos que están tanto en $a como en $b.
$a | $b OR Mantiene los bits prendidos que están en $a o $b o ambos.
$a ^ $b XOR Mantiene los bits prendidos que están en $a o $b pero no en los dos.
~ $a NOT Invierte todos los bits de $a.
$a << $b Corrimiento a la izquierda Corre $a, $b bits a la izquierda (multiplica por $2^{b}$).
$a >> $b Corrimiento a la derecha Corre $a, $b bits a la derecha (divide de forma entera entre $2^{b}$).

Ejemplos prácticos en binario:


$a = 5;  // En binario: 101
$b = 3;  // En binario: 011

echo ($a & $b);  // AND -> Imprime 1  (001)
echo ($a | $b);  // OR  -> Imprime 7  (111)
echo ($a ^ $b);  // XOR -> Imprime 6  (110)

// Corrimientos
$a = 5; 
echo ($a << 2); // Izquierda -> Imprime 20 (10100)

$a = 9; 
echo ($a >> 2); // Derecha   -> Imprime 2  (0010)
7.5 Operadores Lógicos

Al igual que C, PHP usa los mismos operadores lógicos, con una variación en la precedencia de estos dependiendo de cuál se use.

Notación Nombre Resultado
$a and $b AND (Baja prec.) Regresa verdadero (true) si tanto $a como $b son verdaderos.
$a xor $b XOR (Baja prec.) Regresa verdadero (true) si $a o $b es verdadero, pero no los dos.
$a or $b OR (Baja prec.) Regresa verdadero (true) si $a o $b son verdaderos (o ambos).
!$a NOT Regresa verdadero (true) si $a es falso, y viceversa.
$a && $b AND (Alta prec.) Regresa verdadero (true) si tanto $a como $b son verdaderos.
$a || $b OR (Alta prec.) Regresa verdadero (true) si $a o $b son verdaderos.

Nota: Los operadores con símbolo de doble carácter (&& y ||) tienen mayor nivel de precedencia matemática que sus contrapartes en formato de texto plano (and y or).

7.6 Operadores de Comparación

Aquí está el cambio conceptual más fuerte debido al tipado dinámico. Presta especial atención a la diferencia entre comparar solo valores o comparar también tipos.

Operador Significado Descripción
== Igualdad débil. Regresa verdadero si son iguales en valor (hace conversión implícita).
=== Igualdad estricta / fuerte. Regresa verdadero únicamente si son iguales en valor y tipo de dato.
!= o <> Diferente de (débil). Regresa verdadero si las variables difieren en su valor.
!== Diferente de (estricta). Regresa verdadero si difieren en valor o difieren en tipo de dato.
< Menor que. Regresa verdadero si el primer valor es estrictamente menor al segundo.
> Mayor que. Regresa verdadero si el primer valor es estrictamente mayor al segundo.
<= Menor o igual que. Regresa verdadero si el primero es menor o igual al segundo.
>= Mayor o igual que. Regresa verdadero si el primero es mayor o igual al segundo.
<=> Operador Nave Espacial (Spaceship)

Regresa 0 si ambos valores son iguales.

Regresa 1 si el primero es mayor al segundo.

Regresa -1 si el segundo es mayor al primero.

echo 5 <=> 4; // Regresa 1

Nota: Al ser operadores de comparación todos regresan un booleano (true/false), a excepción de la nave espacial (<=>) que devuelve un entero.

7.7 Operadores de Asignación Aritmética

Permiten realizar una operación matemática sobre el valor actual de una variable y guardar el nuevo resultado en ella de forma compacta (idéntico a C).

Operador Equivalente Operación
$a += $b $a = $a + $b Suma y asigna. Se añade el valor de $b a $a.
$a -= $b $a = $a - $b Resta y asigna. Se resta el valor de $b a $a.
$a *= $b $a = $a * $b Multiplicación y asigna. Se multiplica $a por $b.
$a /= $b $a = $a / $b División y asigna. Se divide $a entre $b.
$a %= $b $a = $a % $b Módulo y asigna. Obtiene el residuo de dividir $a entre $b.
$a **= $b $a = $a ** $b Exponenciación y asigna. Eleva $a a la potencia de $b.
7.8 Operadores de Asignación de Bits

Modifican los bits específicos de un entero en memoria de manera directa (idéntico a C).

Operador Equivalente Operación
$a &= $b $a = $a & $b Operación lógica AND a nivel de bits y asigna.
$a |= $b $a = $a | $b Operación lógica OR a nivel de bits y asigna.
$a ^= $b $a = $a ^ $b Operación lógica XOR (OR exclusivo) a nivel de bits y asigna.
$a <<= $b $a = $a << $b Desplazamiento de bits a la izquierda $b veces y asigna.
$a >>= $b $a = $a >> $b Desplazamiento de bits a la derecha $b veces y asigna.
7.9 Operadores de Asignación de Cadenas (Strings)

No existe un operador nativo en C para esto. PHP usa el punto (.) para concatenar.

Operador Equivalente Operación
$a .= $b $a = $a . $b Concatenación y asignación. Pega el string $b al final del string de $a.
8. Estructuras de Control e Importación

Al igual que en C, las estructuras de control alteran el flujo lineal de ejecución del código. PHP hereda casi toda su sintaxis de C, pero añade herramientas potentes para el manejo de colecciones dinámicas y la modularización de archivos.

8.1 Condicionales (if, else, elseif, ternario)

Evalúan una expresión booleana para decidir qué bloque de código ejecutar. En PHP se introduce la palabra clave elseif (todo junto), aunque también se acepta else if (separado), para evitar anidamientos infinitos de llaves.

Ejemplos de uso:

	
<?php
// Ejemplo 1: Condicional básico (idéntico a C)

$edad = 20;
if ($edad >= 18) 
    echo "Es mayor de edad.";
else 
    echo "Es menor de edad.";


// Ejemplo 2: Estructura elseif con evaluación en cadena

$calificacion = 85;
if ($calificacion >= 90) 
    echo "Excelente";
elseif ($calificacion >= 80) 
    echo "Muy Bien"; // Se ejecuta este bloque
else 
    echo "Aprobado";

?>

Además de estos condicionales, PHP nos permite hacer uso del operador ternario que tiene la siguiente sintáxis (condición) ? valor_si_verdadero : valor_si_falso. Esto es simplemente azúcar sintáctica, pero algunas veces nos permite tener código más legible. Veamos un ejemplo de esto: 

	
<?php
	  
$edad = 20;
	  
// Ejemplo con condicional básico if/else

if ($edad >= 18) 
    echo "Es mayor de edad.";
else 
    echo "Es menor de edad.";


// Ejemplo con operador ternario 
echo ($edad >= 18) ? "Es mayor de edad." :  "Es menor de edad.";

?>
8.2 Condicional Múltiple (switch)

Compara una variable o expresión con múltiples casos posibles. Utiliza una comparación débil (==). Recuerda que es fundamental el uso de break al final de cada caso; de lo contrario, la ejecución continuará hacia los casos siguientes de forma automática.

Ejemplos de uso:


<?php
	  
// Ejemplo 1: Control por casos estándar

$rol = 'editor';

switch ($rol) {
    case 'admin':
        echo "Tienes acceso total al sistema.";
        break;
    case 'editor':
        echo "Puedes editar y publicar contenido.";
        break;
    case 'usuario':
        echo "Solo puedes leer el contenido.";
        break;
    default:
        echo "Rol no reconocido. Acceso denegado.";
        break;
}


// Ejemplo 2: Arrastre intencional de casos (sin break intermediario)
$nivel = 3;

switch ($nivel) {
    case 3:
        echo "Acceso VIP activado. ";             //Sin break pasa al caso 2
    case 2:
        echo "+ 100 monedas de bonificación. ";   //Sin break: pasa al caso 1
    case 1:
        echo "+ Llave de bronce obtenida.";       //Fin del switch, no necesita break
}

?>
8.3 Ciclo de pre-prueba (while)

Ejecuta un bloque de instrucciones mientras una condición sea verdadera. La condición se evalúa estrictamente antes de entrar al ciclo; si es falsa desde el principio, el bloque jamás se ejecutará.

Primero pide permiso en lugar de pedir perdón.

Ejemplos de uso:


<?php
	  
// Ejemplo 1: Contador ascendente 
$contador = 1;
while ($contador <= 3) {
    echo "Iteración: $contador \n";
    $contador++;
}


// Ejemplo 2: Condición de parada 
$puntos_vida = 100;
while ($puntos_vida > 0) 
    $puntos_vida -= 10;  // Simulando daño recibido por tormenta en etapa 3
echo "El personaje ha caído.";

?>
8.4 Ciclo de post-prueba (do-while)

A diferencia de while, la condición se evalúa al final de cada iteración. Esto garantiza que el bloque de código dentro del ciclo se ejecutará al menos una vez, sin importar el valor inicial de la condición.

Primero pide perdón en lugar de pedir permiso (lol).

Ejemplo de uso:


<?php	 

// Ejemplo 1: Simulación de un intento de inicio de sesion con contraseña erronea
$intentos = 0;
do {
    $intentos++;
    //Verificación de contraseña...
} while ($intentos < 3);

?>
8.5 Ciclo con contador explícito (for)

Idéntico al de C, es ideal cuando conoces de antemano el número exacto de iteraciones que requieres. Se compone de tres expresiones obligatorias separadas por puntos y comas: for(inicialización; condición; incremento).

Ejemplos de uso:


<?php
	  
// Ejemplo 1: Ciclo controlado tradicional
for ($i = 0; $i < 3; $i++) 
    echo "Índice actual: $i \n";


// Ejemplo 2: Recorrido clásico por tamaño de arreglo 
$numeros = [10, 20, 30];
for ($i = 0; $i < count($numeros); $i++) 
    echo "Valor en posición $i es: " . $numeros[$i] . "\n";

?>
8.6 Ciclo sobre colecciones (foreach)

No existe en C. Es una de las herramientas más potentes y utilizadas de PHP. Está diseñado de forma exclusiva para iterar sobre estructuras complejas como arreglos (arrays) u objetos, abstrayendo por completo el manejo de contadores manuales o índices. Literalmente significa "para cada", y tiene la siguiente sintaxis:

  • foreach (iterable as $valor){...}: Refiere a que por cada elemento en el iterable (algo que se puede recorrer) se ejecute el código de adentro.
  • foreach (iterable as $llave => $valor){...}: Refiere a que por cada elemento en el iterable (algo que se puede recorrer) se extrae de forma explícita tanto su identificador único (clave o índice) como su contenido asociado, y se podrá hacer algo con estos en el código dentro del bloque.

Ejemplos de uso:


<?php
	  
// Ejemplo 1: Lectura directa solo de los valores

$alumnos = ["Ana", "Pedro", "Luis"];
foreach ($alumnos as $nombre) 
    echo "Alumno registrado: $nombre \n";


// Ejemplo 2: Lectura simultánea de Llave (índice) y Valor

$precios = ["Laptop" => 800, "Mouse" => 20, "Teclado" => 50];
foreach ($precios as $producto => $costo) 
    echo "El artículo $producto cuesta \$$costo dólares. \n";

?>
8.7 Interrupción del flujo (break y continue)

Son modificadores de comportamiento para cualquier ciclo (for, while, foreach)

  • break: Interrumpe el ciclo por completo saltando inmediatamente a la instrucción siguiente, es decir, termina el ciclo.
  • continue: Interrumpe la iteración actual y avanza directamente a la siguiente iteración del ciclo.

Ejemplos de uso:


<?php
	  
// Ejemplo 1: Búsqueda rápida con detención total (break)

$nombres = ["Juan", "María", "Objetivo", "Carlos"];
foreach ($nombres as $persona) {
    if ($persona === "Objetivo") {
        echo "¡Encontrado! Deteniendo el ciclo.\n";
        break; // Rompe el foreach por completo
    }
}


// Ejemplo 2: Filtrado u omisión de elementos (continue)

for ($i = 1; $i <= 5; $i++) {
    if ($i === 3) 
        continue; // Se salta el echo para el número 3 y pasa al ciclo 4    
    echo "Número procesado: $i \n";
}

?>
8.8 Interrupción total y/o regreso de valores (return)

Finaliza inmediatamente la ejecución de la función actual y devuelve el control al bloque de código que la invocó, pudiendo opcionalmente regresar un valor como respuesta. Si se ejecuta en el ámbito global (fuera de una función), termina la ejecución del programa completo.

Ejemplos de uso:


<?php

// Ejemplo 1: Regreso básico dentro de una función matemática

function sumar($a, $b) {
    return $a + $b; // Detiene la función y envía el resultado
}
$resultado = sumar(5, 10);


// Ejemplo 2: Salida temprana condicional

function verificarAcceso($edad) {
    if ($edad < 18) 
        return "Acceso denegado."; // Termina la función de inmediato si se cumple
    return "Bienvenido al sistema.";
}

?>
8.9 Importar (include y require)

Permiten utilizar el código completo de un archivo externo dentro de otro antes de que el intérprete ejecute el programa. La diferencia crítica radica en cómo gestionan los errores de archivos no encontrados:

  • include: Emite una advertencia (Warning) y el programa sigue su ejecución.
  • require: Emite un error fatal (Fatal Error) y detiene el programa por completo instantáneamente (Es el estándar seguro de la industria para archivos críticos).

Ejemplos de uso:


<?php
	  
// Ejemplo 1: Carga opcional o de componentes visuales independientes

// Si falta el archivo del pie de página, la aplicación web sigue corriendo.
include 'componentes/footer.php';



// Ejemplo 2: Carga crítica de configuraciones globales

// Si 'conexion.php' no existe, el programa muere inmediatamente por seguridad.
require 'conexion.php'; 

?>
9. Funciones (Definición y Firma)

Las funciones en PHP aíslan bloques de código reutilizables. A diferencia de C, no requieren que se les declare arriba antes de usarlas y se definen usando la palabra clave function sin especificar obligatoriamente el tipo de dato que regresan o reciben. Sigue la convención de nomenclatura: camelCase

Comparación de Firma (C vs PHP):

  • En C: Requiere tipo del valor de regreso, nombre y tipo por cada parámetro así como nombre del parametro: int sumar(int a, int b) { return a + b; }

  • En PHP: Solo usa la palabra clave function. function sumar($a, $b) { return $a + $b; }

Reglas Clave:

  • Los nombres de las funciones NO son sensibles a mayúsculas y minúsculas, a diferencia de las variables. miFuncion() y MIFUNCION() son la misma.
  • Pueden regresar cualquier tipo de dato con return o no regresar nada (equivalente a void en C).
  • PHP permite asignar valores por defecto a los parámetros directamente en la firma.

Ejemplos de uso:


<?php
	  
// 1. Función básica sin parámetros ni valor de regreso

function saludarUsuario() {
    echo "¡Hola, bienvenido de nuevo! \n";
}
saludarusuario(); // Funciona igual (No es sensible a mayúsculas)



// 2. Función con parámetros y valor regreso

function calcularTotal($subtotal, $iva) {
    $total = $subtotal + ($subtotal * $iva);
    return $total;
}
$pagar = calcularTotal(100, 0.16); // pagar = 116



// 3. Función con parámetros con valores por defecto (Opcionales)

// Nota: Los parámetros opcionales siempre van al final de la firma.
function prepararCafe($siguiente, $tipo = "Americano", $azucar = true) {
    if(!siguiente)
        return;
    $mensaje = "Preparando un café $tipo ";
    $mensaje .= $azucar ? "con azúcar." : "amargo.";
    return $mensaje;
}

echo prepararCafe(true);                      // Usa valores por defecto: "Americano con azúcar."
echo prepararCafe(true, "Capuccino", false);  // Sobrescribe ambos: "Capuccino amargo."
?>
10. Arreglos Indexados
10.1 Definición

Recordemos que en C, un arreglo es un bloque continuo de memoria de tamaño fijo y de tipo homogéneo (todos los elementos deben ser del mismo tipo de dato).

En PHP, los arreglos indexados son dinámicos (crecen solos sin preocuparnos por la memoria) y son heterogéneos (pueden guardar un entero, un string y otro arreglo al mismo tiempo). Además, sus posiciones o índices numéricos siempre inician en 0.

Comparativa conceptual:

Característica Arreglos en C Arreglos en PHP
Tamaño Fijo al declarar (ej: int v[5];) Dinámico (Crece bajo demanda)
Tipo de dato Homogéneo (Solo un tipo) Heterogéneo (Tipos mezclados)
Gestión de Memoria Estática o manual (pointers/malloc) Automatizada por el lenguaje

Ejemplo de un arreglo heterogéneo válido en PHP:

<?php
// Guarda un string, un entero y un sub-arreglo
$myArr = array("Volvo", 15, ["manzanas", "platanos"]); 
?>
10.2 Vectores (Arreglos Unidimensionales)

Un vector tiene una sola dimensión (una fila de datos). Veamos cómo cambia la sintaxis de declaración y asignación respecto a C:

¿Cómo lo hacíamos en C? ¿Cómo lo hacemos en PHP? 
// Declaración con tamaño fijo
int numeros[3] = {10, 20, 30};

// Agregar un elemento
//(¡Error! Fuera de rango)
// numeros[3] = 40; 
<?php
	      
// Opción 1: Con Corchetes 
$numeros = [10, 20, 30];

// Opción 2: Con la función array()
$numeros = array(10, 20, 30);

// Agregar elementos es dinámico,
// Se añade al índice [3]
// de forma automática
$numeros[] = 40;
	      
?>

Declaración de Vectores Vacíos:

A diferencia de C, donde necesitas inicializar el tamaño, en PHP puedes crear vectores totalmente vacíos para llenarlos progresivamente:

<?php
	
$datos = [];       // Forma corta
$datos = array();  // Utilizando array()

// Llenado dinámico secuencial
$datos[] = "Primer elemento"; // Índice 0
$datos[] = "Segundo elemento"; // Índice 1
	
?>
10.3 Matrices (Arreglos Multidimensionales)

En C, una matriz se entiende matemáticamente como filas y columnas continuas en memoria (ej: int mat[2][3]). En PHP, una matriz es simplemente un arreglo que contiene otros arreglos dentro.

¿Cómo lo hacíamos en C? ¿Cómo lo hacemos en PHP? 
// Matriz estática de 2x3
int matriz[2][3] = {
    {1, 2, 3},
    {4, 5, 6}
};

// Acceso:
printf("%d", matriz[1][2]);
 
<?php
// Matriz en PHP (Anidando corchetes)
$matriz = [
    [1, 2, 3], // Fila 0
    [4, 5, 6]  // Fila 1
];

// Acceso idéntico a C:
echo $matriz[1][2]; 
?>

Ventaja dinámica de PHP: Las filas no tienen que medir lo mismo obligatoriamente. Una fila puede tener 2 elementos y otra 5, ya que cada fila es un vector independiente.

10.4 Funciones útiles para Arreglos Indexados

PHP nos ahorra hacer ciclos for manuales (como hacíamos en C para buscar, contar u ordenar) gracias a sus funciones nativas:

Función Operación Ejemplo en PHP
print_r() Imprime de forma legible los elementos y sus índices (ideal para debuguear). 
print_r($miArreglo);
array_push() Agrega uno o más elementos al final del arreglo. 
array_push($miArreglo, "naranja", "uvas");
array_pop() Elimina y devuelve el último elemento del arreglo. 
$ultimo = array_pop($miArreglo);
array_unshift() Agrega elementos al inicio, moviendo los índices existentes hacia adelante. 
array_unshift($miArreglo, "rojo", "azul");
array_shift() Elimina el primer elemento del arreglo y lo devuelve. 
$primerElemento = array_shift($miArreglo);
count() Equivalente a calcular el tamaño del arreglo (En C hacíamos sizeof(arr)/sizeof(arr[0])). 
$total = count($miArreglo);
in_array() Busca si un valor existe dentro del arreglo (devuelve true o false). 
$existe = in_array("Azul", $colores);
sort() Ordena los elementos en orden ascendente. 
sort($miArreglo);
11. Arreglos Asociativos
11.1 Definición

En los arreglos indexados (los que ya vimos), las posiciones se identifican con números secuenciales (0, 1, 2...). Un arreglo asociativo nos permite cambiar esos números por claves o llaves personalizadas (módulos de texto o strings) para identificar cada valor. Es decir, es como un diccionario, tiene la palabra que lo identifica y lo que significa: llave => valor

¿Te acuerdas de los struct en C? Cuando queríamos agrupar datos de un alumno (nombre, edad, promedio), creábamos una estructura y accedíamos mediante el punto (alumno.edad). En PHP, un arreglo asociativo cumple exactamente esa misma función, pero sin necesidad de definir un tipo de dato previo.

Concepto En C (Estructuras - struct) En PHP (Arreglos Asociativos)
Definición Rígida. Se debe declarar el struct antes de usarlo. Flexible. Se crea directamente al guardar los datos.
Identificador Miembro/Atributo fijo (ej: prod.precio) Clave de tipo string (ej: $prod["precio"])
Nuevos campos Imposible añadirlos en tiempo de ejecución. Dinámico. Puedes agregar llaves nuevas cuando quieras.
11.2 Declaración y Sintaxis

Para relacionar una llave con un valor en PHP, utilizamos el operador de asignación asociativa => (coloquialmente llamado "flecha gorda").

Ejemplo comparativo de código:

¿Cómo lo hacíamos con un struct en C? ¿Cómo lo hacemos en PHP? 
// 1. Definir la estructura
	      
struct Alumno {
    char nombre[50];
    int edad;
};

// 2. Instanciar y asignar
struct Alumno a1;
strcpy(a1.nombre, "Carlos");
a1.edad = 20;

// Acceso:
printf("%s", a1.nombre);
 
<?php

// Declaración directa con Llave => Valor
$alumno = [
    "nombre" => "Carlos",
    "edad" => 20,
    "aprobado" => true
];

	      
// O usando la función array()
$alumno = array(
    "nombre" => "Carlos",
    "edad" => 20
);


// Acceso (Siempre usa comillas en la clave):
echo $alumno["nombre"]; // Imprime: Carlos
	      
?>

Modificación y Crecimiento Dinámico:

Si quieres modificar un valor o agregar una llave que no existía en la declaración inicial, simplemente la invocas y le asignas el dato: 

<?php
	
$usuario = [
    "username" => "tec_master"
];

// Modificar un valor existente
$usuario["username"] = "dev_php"; 

// Agregar una nueva clave (En C esto rompería el programa)
$usuario["email"] = "correo@correo.com";
	
?>
12.3 Funciones útiles para Arreglos Asociativos

Al no trabajar con índices numéricos, PHP nos provee funciones específicas para administrar, buscar y separar las llaves de los valores:

Función Operación Ejemplo en PHP
array_keys() Extrae solo las llaves del arreglo y devuelve un arreglo indexado con ellas. 
$claves = array_keys($alumno);
// Devuelve: ["nombre", "edad", "aprobado"]
array_values() Extrae solo los valores, ignorando las llaves por completo. 
$valores = array_values($alumno);
// Devuelve: ["Carlos", 20, true]
array_key_exists() Verifica si una clave específica existe dentro del arreglo (devuelve un booleano). 
if (array_key_exists("edad", $alumno)) {
    echo "La edad está registrada.";
}
asort() Ordena el arreglo por sus valores en orden ascendente. 
$precios = ["bici" => 500, "casco" => 80];
asort($precios);
// "casco" seguirá valiendo 80
// y estará primero.
ksort() Ordena el arreglo alfabéticamente por sus llaves, no por los valores. 
ksort($alumno); 
// Ordenará las llaves:
// "aprobado", "edad", "nombre"
